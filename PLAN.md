# miniWebBrowser 計画書

## 0. プロジェクト概要

### ゴール定義

**レベル1ミニブラウザの完成**

- URLを入力するとWebページのテキストを取得・表示できる
- 見出し（h1〜h3）・段落・リンクの区別がつく
- リンクをクリックすると遷移できる
- 通信失敗時にクラッシュしない

### 使用技術スタック

| 種類 | 名称 | 用途 |
|---|---|---|
| 言語 | Python 3.11+ | 全体 |
| GUI | PyQt6 | ウィンドウ表示 |
| HTTP通信 | requests | Webページ取得 |
| HTMLパース | beautifulsoup4 | HTML解析 |
| バージョン管理 | Git / GitHub | コード管理 |

### ファイル構成（最終形）

```
miniWebBrowser/
├── PLAN.md              ← この計画書
├── README.md            ← リポジトリ説明（任意）
├── requirements.txt     ← 使用ライブラリ一覧
├── main.py              ← エントリーポイント
├── browser.py           ← ブラウザ本体（UI含む）
├── fetcher.py           ← HTTP通信担当
├── parser.py            ← HTML解析担当
└── .gitignore           ← Git管理外設定
```

※ Stepが進むにつれてファイルが増えていきます。最初はmain.pyだけです。

### 前提条件（作業開始時点で整っていること）

- GitHubリポジトリがローカルにcloneされている
- Python仮想環境（venv）が作成され、アクティベート済みである
- `pip` が使える状態である
- `.gitignore` が配置されている

5. 動作確認は必ず自分で行う（LLMは動作確認できない）
